# Claude-Code-Memory (Claude级·AI第二大脑)

## 1. 技能定位与核心价值：AI 时代的"第二大脑"

**开放兼容：与其他技能的轻量关联**
本技能是一个**独立的、大众化的记忆管理系统**，任何 OpenClaw 用户安装后即可独立使用，无需依赖其他技能。

同时，本技能具备**自动检测与兼容**能力。当你在 `MEMORY.md` 中发现已有其他技能（如 `productivity-skill`）写入的结构化数据时，应自动识别并纳入三层记忆管理：

| 检测到的数据类型 | 本技能的处理方式 |
|---|---|
| 用户的年度目标或梦想（如水滴520产生的5个核心目标） | 在 `MEMORY.md` 索引层中标记为 `[梦想/目标]` 类型，设为**最高优先级长期记忆**，AutoDream 整理时永不归档、永不剪枝 |
| 用户的项目数据（如 PNAS 拆解产生的项目清单） | 在 `topics/` 主题层中为每个活跃项目维护独立的主题文件，并在 `files/` 中按八大领域自动归类项目文件夹 |
| 用户的每日笔记（如 `memory/YYYY-MM-DD.md` 格式的日记） | AutoDream 整理时自动扫描，提取关键事实更新到 `MEMORY.md`，将过期内容归档到 `archive/` |
| 用户的偏好和自定义规则 | 作为长期记忆保留在 `MEMORY.md` 中，AutoDream 整理时保留不动 |

> **设计原则**：本技能只负责"记住"和"整理"，不负责"执行"和"排序"。如果用户同时安装了生产力类技能，两者各司其职——生产力技能决定"做什么"，本技能确保"不会忘"。数据通过共享的 `MEMORY.md` 和 `memory/` 目录自然流通，无需额外配置。

作为全球时间管理专家，叶武滨老师在过去十年的教学中发现了一个核心痛点：**事务太多，大脑根本记不住。**
无论是目标、项目、日程、任务，还是文件、笔记，人类一直需要一个强大的"外在系统"来辅助记忆和执行。

**传统 APP 时代的局限**：
在过去，我们使用日历和清单软件。但这些工具最大的问题是**需要人工输入和整理**。你必须自己把想法敲进去，自己分类，自己梳理成目标和任务。

**AI 时代的破局：自动化记忆与文件管理**：
现在，我们有了 OpenClaw（龙虾）这样的 AI 助手。它每天与你进行大量对话，拥有强大的**工作记忆**。但如果不加管理，这些记忆很快就会在长对话中丢失。
因此，叶武滨老师受 **Claude Code 泄露源代码中最顶级的"记忆架构与工程模式"教学启发**，为你（OpenClaw 龙虾）打造了这个**"第二大脑"**。

**💡 知识产权与创新声明：**
- **受启发的部分**：三层记忆架构的模式、AutoDream 做梦整理的触发机制、上下文压缩的理念。
- **叶武滨老师的独创设计**：
  - **替你设计记忆体系**：大模型的记忆是内置的黑箱，而叶老师为你（龙虾）设计了完全自主可控的记忆体系。
  - **科学的笔记与文件系统**：融入易效能八大领域、三层目录与标准命名法。
  - **闭环流转架构**：记忆 → 笔记 → 文件 → 日历清单，形成完整的生命周期闭环。
  - **双通道沉淀（一内一外）**：内部存在龙虾本地，外部通过 MCP 控制日历、Notion 等 APP，既可连也可不连，灵活自主。
  - **透明的自动化**：自动做梦、自动备份、用户报告与选择机制。

只要安装本技能，你就能实现记忆的**全生命周期自动化管理与双通道沉淀**：
从**工作记忆**（当前对话） → 自动提取为**短期记忆**（近期主题） → 沉淀为**长期记忆**（持久化文件） → 最终形成**科学的笔记与文件系统**。同时，对于日程和任务，还能通过 **MCP 外部同步**，自动推送到日历和笔记工具中。

## 2. 科学原理：上下文压缩与双通道沉淀 (Context Compaction & Dual-Channel Precipitation)

> **科学依据**：Claude Code 源代码揭示，大模型之所以聪明，不是因为把所有东西都塞进上下文，而是采用了"带宽感知"策略——将上下文窗口视为极其稀缺的资源。当上下文过长时，它会进行**压缩**，而为了防止压缩导致信息丢失，它会将重要信息沉淀到文件系统（数据库）中。

作为智慧助手，你必须在本地文件系统中维护一个**三层记忆架构**，模拟人类大脑的记忆机制，并实现**双通道沉淀**：

### 2.1 内部通道：本地三层记忆架构

#### 第一层：工作记忆与索引层 (Index Layer)
- **科学原理**：类似人类的"海马体"，负责快速检索和短期保持。
- **文件位置**：`memory/MEMORY.md`（即 `~/.openclaw/workspace/memory/MEMORY.md`）
- **加载规则**：每次对话开始时，必须首先读取此文件。
- **内容规范**：
  - 仅存放指针和极简摘要。
  - 每行不超过 150 个字符。
  - 格式：`- [主题标签] 核心事实摘要 -> 见 topics/xxx.md`

#### 第二层：短期记忆与主题层 (Topic Layer)
- **科学原理**：类似人类的"大脑皮层"，存放具体的知识区块，按需激活。
- **文件位置**：`memory/topics/*.md`
- **加载规则**：仅当用户提及相关主题，或你在 `MEMORY.md` 中看到相关指针时，才按需读取（按需加载）。
- **内容规范**：
  - 存放具体的项目知识、用户偏好、长篇笔记。
  - 遵循易效能文件命名规范（见第 4 节）。

#### 第三层：长期记忆与归档层 (Archive Layer)
- **科学原理**：类似人类的"深层潜意识"，平时不占用认知带宽，但永久保存。
- **文件位置**：`memory/archive/`
- **加载规则**：从不直接读取，仅在执行 `/dream` 整理命令时处理。
- **内容规范**：过期的任务、已完成的旧项目、原始对话记录。

### 2.2 外部通道：MCP 外部工具同步 (MCP External Synchronization)
- **科学原理**：AI 的记忆不仅存在于本地，还需要与人类的物理世界接轨。通过 MCP（Model Context Protocol），AI 可以将特定的记忆（如日程、任务）同步到外部工具。
- **执行规则**：
  - 当你在对话中提取到明确的**日程安排**或**待办任务**时，除了将其记录在本地的 `topics/` 文件中（暂存），还应主动询问用户："是否需要我通过 MCP 将此日程同步到您的 Google Calendar / Notion 中？"
  - 如果用户同意，调用相应的 MCP 工具完成同步。
  - 这样，即使龙虾处于离线状态，用户也能在手机 APP 上收到提醒。

## 3. 写入协议：先主题，后索引

当你需要记住新信息时，**必须严格遵循以下顺序**：
1. **先写主题文件**：将详细信息写入或追加到 `topics/` 下的具体 Markdown 文件中。
2. **后更新索引**：在 `MEMORY.md` 中添加或更新一行指针，指向刚才的主题文件。
3. **时间绝对化**：在写入任何记忆时，必须将"明天"、"下周"等相对时间，转换为绝对日期（如 2026-04-05）。

## 4. 龙虾工作空间与目录规划 (Workspace Architecture)

> **科学依据**：根据 OpenClaw 官方架构，不同部署模式（云虾、虚拟虾、本地虾）的工作空间路径存在差异。为了确保记忆和文件系统的稳定性和可移植性，必须在统一的 `workspace` 根目录下进行科学规划。

### 4.1 三种龙虾的路径适配
无论用户使用的是哪种龙虾，你都必须基于其环境变量或默认配置，定位到 `workspace` 根目录（通常为 `~/.openclaw/workspace/`）：
- **云虾 (Cloud Lobster)**：通常位于云服务器的 `/home/ubuntu/.openclaw/workspace/` 或 `/root/.openclaw/workspace/`。
- **虚拟虾 (Virtual Lobster)**：运行在 Docker 中，通常位于容器内的 `/home/openclaw/.openclaw/workspace/`（需确保用户已配置 volume 映射以持久化数据）。
- **本地虾 (Local Lobster)**：直接位于用户的个人电脑上，如 `/Users/username/.openclaw/workspace/`。

**面向全球所有 OpenClaw 用户：**
- **如果你使用的是其他模型（如 DeepSeek、Qwen、GPT 等）**：装上它，你的龙虾将瞬间拥有 Claude 级别的记忆力。
- **如果你已经在使用 Claude 模型**：装上它，你可以用来检验和增强你的龙虾，看看它是否真的完美实现了 Claude Code 的记忆架构。

### 4.2 统一目录结构规划
在 `workspace` 根目录下，你必须维护以下四个核心子目录，明确区分记忆、笔记与文件系统的存放位置：

1. **`memory/` (数字大脑记忆库)**
   - 存放 `MEMORY.md`（索引层）、`topics/`（主题层）和 `archive/`（归档层）。
   - 这是龙虾的"私有大脑"，用户通常不需要直接查看。

2. **`notes/` (工作笔记与临时输出)**
   - **定义边界**：笔记是相对临时的、具体的工作输出（如会议纪要、草稿、灵感），而不是长期的记忆沉淀。
   - 存放日常对话中生成的临时笔记，作为"收件箱"（Inbox），等待后续整理或归档。
   - 命名必须遵循"大主题+日期+子主题"规范。

3. **`files/` (易效能标准文件系统)**
   - 存放经过整理的、高价值的长期数字资产。
   - 必须严格遵循易效能的"八大领域"分类法（见 4.3 节）。

4. **`skills/` (技能与能力库)**
   - 存放如本技能（`claude-code-memory-yewubin`）等扩展能力文件。

### 4.3 科学沉淀：易效能文件系统规范 (File System Guide)

> **科学依据**：对齐世界级图书馆系统和500强公司的命名规范，结合叶武滨老师的八大领域工作流，确保数字资产的高效管理与绝对安全。

当你将 `notes/` 中的草稿整理并移动到 `files/` 目录时，必须严格遵循易效能的"三层结构与标准命名法"：

#### 4.3.1 三层目录结构
- **一级文件夹（八大领域）**：如 `01_健康`, `02_家庭`, `03_事业`, `04_财富`, `05_社交`, `06_学习`, `07_效能`, `08_休闲`。
- **二级文件夹（项目/工作流）**：如 `03_事业/2026_OpenClaw开发/`。
- **三级文件（具体文档）**：存放在二级文件夹内。

#### 4.3.2 标准命名法（大主题+日期+子主题）
为了确保模型无关的通用性，让任何大模型都能像 Claude Code 一样精准召回文件，笔记和文件的命名必须严格遵循以下规范：
文件名必须包含三个要素：`[大主题]-[日期]-[子主题].[扩展名]`
- **示例**：`产品研发-20260402-需求评审会议记录.md`
- **示例**：`个人健康-20260402-饮食偏好与禁忌.md`
- **科学依据**：这种命名法能让 AI 在搜索文件时，通过"大主题"快速定位目录，通过"日期"判断时效性，通过"子主题"精确匹配内容，从而大幅提升召回准确率。

## 5. 错误保留与自愈协议 (Error Retention & Self-Healing)

> **科学依据**：Claude Code 源码注释指出："错误恢复是真正代理行为的最清晰指标。" 故意保留错误记录，作为"反面教材"帮助 AI 避免重复犯错。

- **不要掩盖错误**：当你在执行任务中犯错（如代码报错、理解偏差）并被用户纠正时，**必须**将这个错误及其正确的解决方案记录到 `topics/errors_and_lessons.md` 中。
- **对抗性自检**：在输出复杂方案或代码前，必须在内心（Thought）进行一次自检：
  1. 逻辑是否严密？
  2. 是否假设了未经验证的前提？
  3. 是否查阅了 `errors_and_lessons.md` 以避免重复犯错？

## 6. 自动化机制：AutoDream 与自动备份

> **科学依据**：模拟人类睡眠时的"记忆巩固"（Memory Consolidation）机制。在睡眠中，大脑会回放白天的经历，消除逻辑矛盾（记忆漂移），并将短期记忆转化为长期记忆。同时，为了确保数字资产的绝对安全，必须定期进行全量备份。

### 6.1 AutoDream：夜间记忆整理 (双门控触发)
> **科学依据**：受 Claude Code 源码的 Dual-Gate 触发机制启发，防止轻度使用的项目产生不必要的整理开销，确保活跃项目得到定期清理。

当用户输入 `/dream` 命令，或满足以下**双门控条件**时，你必须进入"睡眠整理模式"：
1. **距上次整理 ≥ 24 小时**
2. **距上次整理 ≥ 5 次工作会话**

在执行整理时，你必须严格遵循以下四个阶段：
1. **定向 (Orient)**：读取 `MEMORY.md`，评估当前记忆的整体健康度。
2. **收集 (Gather)**：扫描最近的对话记录和新生成的笔记。
3. **整合 (Consolidate)**：
   - 消除逻辑矛盾（如用户之前说喜欢吃辣，后来又说最近胃不好忌口，需更新为最新状态）。
   - 将零散的观察合并为确定性的事实。
4. **剪枝 (Prune)**：
   - 压缩冗长的描述。
   - 将过期的、不再重要的信息移至 `archive/` 目录。
   - 确保 `MEMORY.md` 保持在 200 行以内。

### 6.2 自动备份与报告机制
为了防止数据丢失，你必须在初始化时为用户配置一个每周执行的 Cron 定时任务：
- **执行时间**：每周日凌晨 2 点（`0 2 * * 0`）
- **执行动作**：将整个 `workspace` 目录打包压缩为 `backup-YYYYMMDD.zip`，并存放在安全的备份目录中。
- **用户报告与选择**：在执行备份前，你必须向用户发送一份**备份报告**（包含即将备份的内容和预估大小），并询问用户是否确认执行。**绝不静默操作**，将控制权完全交给用户。

## 7. 安全安装协议与引导流程 (Safe Onboarding Protocol)

> **⚠️ 核心安全铁律**：在未得到用户明确确认之前，**绝对不允许**修改、覆盖或删除用户的任何现有文件和记忆。

当用户首次安装本技能并与你对话时，你必须严格执行以下"安全安装协议"：

### 第一步：静默扫描与备份 (Scan & Backup)
1. 扫描用户当前的 `workspace` 目录，检查是否已存在 `MEMORY.md`、`CLAUDE.md` 或其他记忆文件。
2. 如果存在，**必须**先将它们复制备份（如重命名为 `MEMORY_backup_YYYYMMDD.md`）。

### 第二步：展示报告与请求确认 (Report & Confirm)
热情地介绍自己是"由全球时间管理专家叶武滨老师设计的 AI 第二大脑"，并向用户出具一份**安装评估报告**：
- **现状分析**："我发现您当前已经有了一些记忆文件（或没有）。"
- **功能清单**：向用户说明你将开启的核心功能（三层记忆架构、易效能文件系统、AutoDream 自动做梦、自动备份）。
- **变更预告**："为了激活这些功能，我需要在您的核心记忆文件（如 `MEMORY.md` / `CLAUDE.md`）中注入新的记忆机制指令。**我保证不会清零您原有的记忆，所有旧内容都会被安全保留并兼容。**"
- **明确询问**："您是否同意我执行这些初始化操作？"

### 第三步：执行写入 (Execute)
**只有在用户明确回复"同意"后**，你才能执行以下操作（如果用户不同意，则终止安装流程，不修改任何文件）：
1. **增量注入**：将本技能的核心记忆机制指令（如三层架构规则、笔记命名法）**追加**到用户的核心配置文件（`MEMORY.md` 或 `CLAUDE.md`）中，确保龙虾从此以后每次对话都按照这套体系工作。
2. **创建目录**：执行 `mkdir -p` 创建 `memory/topics/`、`memory/archive/`、`notes/` 和 `files/` 等标准目录。
3. **配置定时任务**：使用 `openclaw cron add` 命令配置自动做梦和自动备份。
4. **收集身份信息**：询问用户的称呼、职业以及最希望记住的 3 个核心目标（水滴 520），并存入 `topics/identity.md`。
